## Детальное описание этапов

### Этап 1: Вход в локацию

**Цель:** Первичная разведка и оценка обстановки

**Действия игрока:**
- Перемещение по 3D-пространству серверной
- Визуальный осмотр стоек, шкафов, рабочих мест
- Поиск незапертых терминалов или открытых портов
- Обнаружение заметок, стикеров с паролями, схем сети

**Риски:**
- Охрана (патрули, камеры наблюдения)
- Датчики движения
- Превышение времени нахождения в зоне
- Шум (громкие действия могут привлечь внимание)

**Инструменты:**
- Бинокль/камера для удалённого осмотра
- Сканер Wi-Fi для обнаружения сетей
- Тепловизор для поиска активного оборудования

### Этап 2: Подключение к терминалу/стойке

**Цель:** Получить доступ к системе

**Физические действия:**
- Найти свободный порт коммутатора
- Подключить патч-корд (мини-игра на точность)
- Вставить загрузочный носитель (USB, CD, дискета)
- При необходимости — сбросить питание (hard reset)

**Запуск сессии:**
- Загрузка собственного ОС (LiveCD, USB)
- Или подключение к существующей системе
- Проверка доступных интерфейсов

**Загрузка инструментов:**
- Выбор скриптов из инвентаря
- Проверка совместимости с целевой системой
- Настройка параметров подключения

### Этап 3: Выполнение задачи

#### Тип A: Взлом целевого узла (Red Team)

**Подзадачи:**
1. Сканирование сети (`scan`)
2. Построение маршрута (`route`)
3. Обход фаервола (`bypass`)
4. Дешифровка данных (`decrypt`)
5. Извлечение информации (`extract`)

**Механики:**
- Пошаговое продвижение по узлам сети
- Решение головоломок на каждом узле
- Управление риском обнаружения

#### Тип B: Аудит конфигурации (Blue/Grey Team)

**Подзадачи:**
1. Анализ текущих настроек
2. Выявление уязвимостей
3. Применение патчей (`patch`)
4. Настройка правил фильтрации
5. Документирование изменений

**Механики:**
- Проверка списков конфигурации
- Сравнение с эталонными настройками
- Применение исправлений без нарушения работы

#### Тип C: Отражение атаки (Blue Team)

**Подзадачи:**
1. Обнаружение аномалий в логах
2. Идентификация источника атаки
3. Изоляция заражённых сегментов
4. Контрмеры против эксплойтов
5. Восстановление после инцидента

**Механики:**
- Реальное время с нарастающей угрозой
- Приоритизация задач при ограниченных ресурсах
- Баланс между uptime и безопасностью

### Этап 4: Управление ресурсами

**Типы ресурсов:**

| Ресурс | Описание | Лимит | Восстановление |
|--------|----------|-------|----------------|
| **Вычислительные циклы** | Лимит команд/скриптов за сессию | Зависит от оборудования | После завершения миссии |
| **Пропускная способность** | Скорость передачи данных | Зависит от соединения | Мгновенно, но влияет на обнаружение |
| **Уровень обнаружения** | Риск быть замеченным | 0–100% | Снижается со временем или действиями |
| **Время** | Ограничение сессии | Зависит от миссии | Не восстанавливается |

**Стратегии управления:**
- Быстрые агрессивные действия → высокое обнаружение, мало времени
- Медленные скрытные действия → низкое обнаружение, больше времени
- Балансированный подход → средние показатели по всем параметрам

### Этап 5: Завершение сессии

**Критерии успеха:**
- Основная цель выполнена
- Уровень обнаружения не превышен
- Критические системы не повреждены (для Blue Team)

**Награды:**

| Тип награды | Описание | Использование |
|-------------|----------|---------------|
| **Данные** | Конфиденциальная информация, логи, ключи | Продажа, сюжет, крафт |
| **Репутация** | Рейтинг в сообществе | Доступ к новым заказчикам |
| **Чертежи** | Схемы оборудования, спецификации | Улучшение снаряжения |
| **Фрагменты сюжета** | Письма, записи, диалоги | Открывают новую информацию |
| **Кредиты** | Внутриигровая валюта | Покупка оборудования, скриптов |

**Отчёт о миссии:**
- Статистика выполнения
- Допущенные ошибки
- Бонусы за стиль игры
- Разблокированные достижения

                         │

**Категории улучшений:**

1. **Скрипты и программы**
   - Новые команды терминала
   - Ускоренные версии существующих скриптов
   - Специализированные инструменты

2. **Аппаратные модули**
   - Более быстрые сетевые карты
   - Увеличенные накопители
   - Аппаратные дешифраторы

3. **Пассивные бонусы**
   - Снижение уровня обнаружения
   - Увеличение лимитов ресурсов
   - Автодополнение и подсказки

**Стоимость улучшений:**
- Зависит от редкости и мощности
- Требует определённого уровня репутации
- Некоторые улучшения требуют чертежей

## Варианты игровых режимов

### Кампания
- Последовательная цепочка миссий
- Раскрытие основного сюжета
- Фиксированные награды и прогресс

### Песочница
- Свободный выбор сценариев
- Настройка параметров перед началом
- Без влияния на основной прогресс

### Ежедневные контракты
- Случайная генерация задач
- Уникальные модификаторы
- Лидерборды и специальные награды

### Обучение
- Пошаговое введение в механики
- Безопасная среда для экспериментов
- Подсказки и ограничения на ошибки
